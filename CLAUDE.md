# Architecture Explorer - Claude Code Guide

## What This Is
An interactive architecture visualization tool for the Rival Insurance Technology Rating Platform. It shows how all backend components connect, their build status, real activity from Azure DevOps, and AI-generated insights.

**GitHub repo:** https://github.com/muhammadut/rival-tracker (private)

## Files
- `index.html` - The main visualization (pure HTML/CSS/JS, zero dependencies)
- `data-base.js` - Static architecture data (nodes, edges, flows, status, meta)
- `data.js` - Enriched data with live ADO activity (generated by enrich-data.js)
- `enrich-data.js` - Script that pulls real data from Azure DevOps API + local git repos
- `insights-data.js` - AI-generated code review insights (generated by Claude Code)

## How to Refresh Data

### Step 1: Pull fresh ADO data
```bash
export ADO_PAT='<your-pat-here>'
cd /Users/tariqusama/Documents/azure_devops/architecture-explorer
node enrich-data.js
```
This pulls commits, PRs, work items, and builds from Azure DevOps.

### Step 2: Generate insights (Claude Code does this)
Tell Claude Code:
> "Refresh the architecture explorer insights. Launch sub-agents to review each key repo (QuoteManagement, RatingOrchestrator, CarrierConnector, SchemaCache, Platform.Web, Platform.BFF) for code quality issues, integration risks, deployment gaps, and architectural concerns. Write findings to insights-data.js."

Claude Code will:
1. Launch sub-agents that read the actual source code in each repo
2. Cross-reference against the architecture diagram connections
3. Look for: empty configs, commented-out code, hardcoded values, missing error handling, bus factor risks, integration gaps, deployment blockers
4. Write structured findings to `insights-data.js` with severity levels (CRITICAL/WARNING/INFO)

### Step 3: Push updates
```bash
cd /Users/tariqusama/Documents/azure_devops/architecture-explorer
git add -A && git commit -m "Refresh data and insights" && git push
```

### Step 4: View
Open `index.html` in a browser.

## How to Update the Architecture

If new services are added or the architecture changes:
1. Edit `data-base.js` to add/modify nodes, edges, and meta
2. Run `node enrich-data.js` to regenerate `data.js` with fresh enrichment
3. Tell Claude Code to regenerate insights
